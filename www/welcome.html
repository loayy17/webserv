Server Features

Everything implemented from scratch, RFC-compliant and stress-tested

⚡

Non-Blocking I/O

Single poll() loop handles thousands of concurrent connections. No thread per connection — all state machines driven by events.

🔑

Session Management

Cookie-based sessions with cryptographically random IDs from /dev/urandom. Automatic expiry & periodic cleanup.

📄

CGI Support

Execute Python, PHP or any interpreter via cgi_pass. Full environment variables, PATH_INFO, body streaming and chunked decoding to CGI stdin.

📥

File Uploads

Multipart form-data and raw-binary POST uploads. Filename sanitisation, configurable client_max_body_size per location, auto-creates upload directories.

📁

Directory Listing

Beautiful auto-index pages with file icons, sizes and last-modified dates. Toggle per location with autoindex on/off.

🔁

HTTP Redirects

Permanent (301) and temporary (302) redirects configured with the return directive, identical to NGINX syntax.

📞

Virtual Hosts

Multiple server {} blocks sharing the same port. Request routing by Host: header with sensible default fallback.

🗎

Chunked Transfer

Full RFC 7230 chunked request decoding — hex chunk sizes, optional extensions, and trailer field support — before handing body to CGI or upload handler.

🌁

Custom Error Pages

Per-server or per-location error_page directive. Stunning animated built-in fallback pages for all 4xx/5xx codes.

🚫

Method Control

Whitelist allowed methods per location — GET, POST, DELETE, HEAD. Returns 405 Method Not Allowed with proper Allow: header.

📈

Keep-Alive

HTTP/1.1 persistent connections. Per-request state is reset after each response so the same socket can serve hundreds of sequential requests.

⏳

Timeout Handling

Idle client & CGI process timeouts (configurable). Timed-out CGIs receive SIGTERM then SIGKILL and the client gets a 504 Gateway Timeout.

webserv.conf — annotated example 
http {
    # Global body-size limit (overridable per server/location)
    client_max_body_size 5M;

    server {
        listen       0.0.0.0:8080;
        server_name  localhost;
        root         ./www;
        index        welcome.html;

        # Custom error pages (optional)
        error_page   404  ./www/errors/404.html;
        error_page   500  ./www/errors/50x.html;

        # Root static location
        location / {
            root                ./www;
            methods             GET POST DELETE;
            autoindex           on;
            client_max_body_size 5M;
        }

        # File-upload endpoint
        location /upload {
            root                ./www;
            methods             GET POST;
            upload_dir          ./uploads;
            client_max_body_size 10M;
        }

        # Python CGI
        location /cgi-bin {
            root    ./www/cgi-bin;
            methods GET POST;
            cgi_pass .py  /usr/bin/python3;
        }

        # HTTP redirect
        location /old-page {
            return 301 /;
        }

        # DELETE demo (browse + delete uploaded files)
        location /delete {
            root      ./uploads;
            methods   GET DELETE;
            autoindex on;
        }
    }

    # Second virtual host on a different port
    server {
        listen      0.0.0.0:8081;
        server_name second.localhost;
        root        ./www;
        index       welcome.html;
        client_max_body_size 1M;

        location / {
            root    ./www;
            methods GET;
        }
    }
}

Quick Start

From clone to running server in under a minute

1

Clone & Build

git clone <repo> && cd webserv && make

2

Create or edit a config file

Copy webserv.conf as a starting point and adjust paths, ports and CGI interpreters.

3

Run the server

./webserv webserv.conf  —  defaults to default.conf if no argument is given.

4

Open in your browser

Navigate to http://localhost:8080/ and explore every feature from this page.

5

Stop gracefully

Press Ctrl-C — the server handles SIGINT / SIGTERM and cleans up all connections.
